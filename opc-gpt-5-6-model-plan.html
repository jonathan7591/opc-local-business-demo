GPT‑5.6 三模型分工建议

目标：给当前海外本地商家获客 / demo / 外联工作分配更合适的模型，同时不触碰默认 profile、飞书、企业微信或微信的现有路由。

结论：不需要改 default。现有 OPC profiles 已经天然隔离：opc-ceo 用 Sol、opc-sales 用 Terra、opc-worker 用 Luna。后续只在这些 profile 或其 cron job 里按任务指定模型即可。

三个模型：已知配置 + 同题实测

gpt-5.6-sol

Sol：决策与策略

现有配置：opc-ceo

同题表现：最简洁地指出 offer、外联角度与“对方可能已有隐藏预约工具”的验证风险。

建议工作：地区/行业取舍、ICP 定义、报价、offer 设计、是否继续推进某一商家。

不建议：大量低价值候选整理、机械抓取、逐家生成代码。

gpt-5.6-terra

Terra：主力交付

现有配置：default、opc-sales 及多名 OPC worker

同题表现：给出最完整的 booking funnel offer、明确 CTA（包含 deposit / WhatsApp）和询盘前验证条件。

建议工作：商家复核、官网/社媒分析、demo 文案、外联草稿、落地页/代码实施、交付 QA。

不建议：改 default 来给其他平台“强行统一”。

gpt-5.6-luna

Luna：低成本初筛

现有配置：opc-worker，low reasoning，terminal/messaging 已禁用

同题表现：能完成基础 offer/风险判断，额外提醒合规；整体更适合规则明确、可复核的任务。

建议工作：候选分类、去重、字段补全、官网/社媒初筛、机会评分、生成待人工审核队列。

不建议：最终商家外联、报价、重要策略或未经审核的公开内容。

当前 OPC 推荐流水线

阶段负责 profile / 模型输出是否能直接对外发送
1. 候选发现与结构化opc-worker · Luna候选清单、去重、公开联系渠道、初步评分不能
2. 商机判断opc-ceo · Sol是否值得做 demo、核心痛点、报价/offer不能
3. Demo / 文案 / 外联草稿opc-sales · Terra落地页 brief、英文邮件、contact-form 文案、跟进方案不能
4. 最终审稿opc-reviewer · Claude Sonnet 5事实、品牌、合规、CTA 与拼写检查不能
5. 人工批准后发送当前主会话 / approved send单家发送、状态记录、回复跟进仅在你明确批准后

隔离原则：不会影响飞书、企业微信、微信

Hermes 官方 profile 机制使每个 profile 拥有独立的 config.yaml、.env、memory、sessions、skills 和 gateway state。当前机器也已经有独立的 opc-ceo、opc-sales、opc-worker profiles。

本方案不做的事：

  • 不改 /home/admin/.hermes/config.yaml 中的 default: gpt-5.6-terra
  • 不改飞书、企业微信、微信的 token、platform 配置或 routing
  • 不重启默认 gateway
  • 不把 OPC 的 persona、memory 或任务带进其他平台会话

只做的事：通过 hermes -p opc-ceo …、hermes -p opc-sales …、hermes -p opc-worker … 或给 OPC 专用 cron job 设置 model override 来运行工作任务。

建议的下一步（仍然零配置改动）

  1. Luna 先整理 Salford / Stockport 商家候选，并标记 Google Maps 必须复核项。
  2. Sol 只判断前 10 家里哪 3 家值得做 demo，避免做一堆没人看的页面。
  3. Terra 为通过的 3 家做针对性 demo 和外联草稿。
  4. Reviewer 审核后，仍由你逐家批准发送。

注意：Terra / Sol / Luna 是当前 custom provider 暴露的模型标识；本报告的能力判断来自本机实际配置及同一条无工具 prompt 的隔离小测，并不是供应商公开规格或跑分宣传。
